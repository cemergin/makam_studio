# Instrument — qanun

This is a Phase 1 placeholder. Content will migrate / reference from `~/lab/makam_studio/research/qanun-instrument-design.md` (wave-1) and `qanun-makers-and-luthiery.md` (wave-2), per the recommendation in `research/beatforge-reusable-research-mining.md`.

Planned content:

- Physical construction (~78 strings = 26 courses × 3, mandal lever mechanism, soundboard, bridges, plectra).
- Mandal mechanics — Turkish vs Arabic mandal density and stops (Turkish ~12 stops/course supporting AEU 24-EDO comma steps; Arabic typically fewer stops focused on quarter-tones).
- Playing technique — fingering, plectrum positioning per string-course, the mandal-flip-during-playing tradition.
- Maker lineages — Ali Eken, Ejder Güleç, Nahat lineage. Materials science. Pricing tiers.
- The "qanun click" — the audible sound a mandal makes when flipped. Some users consider it part of the instrument's voice; others find it kitsch (per `docs/design-direction.md` §"Open questions").
- Control-surface mapping — how the physical instrument's affordances translate to the on-screen UI (per the design-direction.md §pivot 1: piano = canonical; qanun = honored model).

There is no direct BeatForge analog — drum machines don't have an equivalent depth-of-instrument doc. This folder is wholly original work.
